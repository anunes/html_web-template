Everything you need to know to go from this template to a live website. No coding experience required for the basics.

Project Structure

Here is every file in the project and what it does:

index.html ← your pages live here ✏️
assets/
  css/
    style.css ← YOUR custom styles go here ✏️
  js/
    config.js ← change ALL text & links here ✏️
  img/ ← drop your images here ✏️
engine/ ← do not edit anything in here
  css/app.css ← template styles
  js/app.js ← template logic
  js/components/ ← building blocks
Golden rule: The only files you ever need to edit are index.html, assets/js/config.js, and assets/css/style.css. Everything inside engine/ runs the template — leave it alone.

Step-by-step Guide

  1. Change the site name

    Open assets/js/config.js and update the brand value:

    navbar: {
  brand: 'My Awesome Site',   // ← change this
  ...

    This updates the logo text in the top-left of the navbar instantly.

  2. Edit navigation links

    Still in config.js, update the links array. Each item needs a label (what the user sees) and an href (where it scrolls to — must match the page-id of your page).

    links: [
  { label: 'Home',      href: '#home'      },
  { label: 'Portfolio', href: '#portfolio' },
  { label: 'Contact',   href: '#contact'   },
],

  3. Edit or remove the dropdown menu

    The dropdown section in config.js controls the Services menu. Change labels and links, add a { divider: true } item for a separator, or set dropdown: null to remove it entirely.

    dropdown: {
  label: 'Products',
  items: [
    { label: 'App',     href: '#app'     },
    { label: 'Plugin',  href: '#plugin'  },
    { divider: true },
    { label: 'Pricing', href: '#pricing' },
  ],
},

  4. Update the footer

    At the bottom of config.js, update the footer block. Change yearStart to the year your project began, and domain to your website domain. Set domain: '' to hide it.

    footer: {
  yearStart: 2024,
  domain:    'mysite.com',
},

  5. Add a new page

    1. Add a nav link in config.js:

    { label: 'Portfolio', href: '#portfolio' }

    2. Add an <app-page> element inside <main class="main-container"> in index.html:

    <app-page page-id="portfolio" title="My Portfolio">
  <p>Your content goes here.</p>
</app-page>
    Important: The page-id must exactly match the #id used in the nav link href (without the #).

  6. Style your site

    Open assets/css/style.css — this is your blank canvas. Use standard CSS. To quickly change colours, override a CSS variable:

    :root {
  --accent-color: #e63946;  /* links, buttons, active nav */
  --nav-bg:       #ffffff;  /* navbar background          */
}

    All available variables are listed as comments at the top of style.css.

  7. Deploy your site

    No build step needed — host it anywhere for free:

    • GitHub Pages — push to a repo, enable Pages in Settings.
    • Netlify — drag and drop your project folder on netlify.com.
    • Vercel — connect your GitHub repo on vercel.com.
    Note: ES modules require the page to be served over HTTP — not opened as a local file. Run npx serve . in your project folder to test locally before deploying.
